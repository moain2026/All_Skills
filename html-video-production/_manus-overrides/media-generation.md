# Manus-native Media Generation Contract

This file is the single source of truth for **how speech, music, images, and audio transcription are produced inside the `html-video-production` skill in Manus**. It overrides any third-party CLI / API instructions that may still appear in upstream HyperFrames reference files. Whenever upstream content describes a different path (Kokoro, `npx hyperframes tts`, `npx hyperframes transcribe`, ElevenLabs, HeyGen TTS, Azure, etc.), the contract below wins.

## Hard rules

1. **Narration / voiceover audio** — produce with the Manus `generate_speech` tool. Never call any third-party TTS path.
2. **Background music / underscore / jingles** — produce with the Manus `generate_music` tool. Never compose music with ffmpeg, hand-write samplers, or call third-party music APIs.
3. **Raster images / icons / product photos / textured backgrounds** that are not provided by the user — produce with the Manus `generate_image` tool (or `generate_image_variation` for edits to existing images). Never use placeholder image services or assume "the image will appear at runtime".
4. **Word-level timestamps for any audio** (whether generated by `generate_speech` or supplied by the user) — obtain via the OpenAI Whisper API with `timestamp_granularities=["word"]`. Never run `npx hyperframes transcribe`, whisper.cpp, or Groq Whisper API.
5. **Always generate the asset before referencing it** in HTML. Do not write `<audio src="narration.wav">` or `<img src="logo.png">` against a path that does not yet exist on disk.

## Routing

The composition workflow uses these assets in three places:

| Need | Path |
| --- | --- |
| Spoken narration / VO | `generate_speech` |
| Background music / underscore | `generate_music` |
| Raster images / icons | `generate_image` |
| Audio that needs word timestamps | OpenAI Whisper API (`timestamp_granularities=["word"]`) |
| `.srt` / `.vtt` cues supplied by user | Read file directly, parse cues into `[{ text, start, end }]` |

Read the [`tts-prompter`](https://github.com/manus-ai/skills) for the speech-side prompt construction (voice selection, SSML pacing, duration control). Read the [`music-prompter`](https://github.com/manus-ai/skills) skill for the music-side prompt construction. They are the upstream references for prompt quality; this document is the integration contract for slotting their outputs into a HyperFrames project.

## Speech generation contract

### Step 1 — Decide voice and language

Before calling `generate_speech`, pick:

- **Voice** — choose by content type and target audience. The `tts-prompter` voice catalog is authoritative.
- **Language** — match the spoken script. SSML can mix languages mid-utterance via `<lang xml:lang="...">`.
- **Speaking rate** — control with SSML `<prosody rate="...">`. Default is natural pace; use `slow` / `medium` / `fast` or explicit percentages.

Do not depend on Kokoro voice IDs (`af_*`, `bf_*`, etc.) — those are upstream and not addressable here.

### Step 2 — Write `narration.txt` first, then synthesize

Always materialize the spoken text as a separate file before calling the tool:

```
narration.txt    ← the exact string passed to generate_speech, with pronunciation
                   substitutions applied (e.g. "API" → "A P I", "$2T" → "two trillion")
SCRIPT.md        ← the human-readable creative doc, distinct from narration.txt
```

This makes regeneration with a different voice trivial.

### Step 3 — Call `generate_speech` with SSML

Use SSML by default, not plain text. SSML gives pacing control:

- **`<break time="...ms"/>`** for explicit pauses that match storyboard pacing.
- **`<prosody rate="..." pitch="..." volume="...">`** for energy shifts.
- **`<emphasis level="...">`** for stressed words.

Example SSML structure for a beat-driven narration:

```xml
<speak>
  Hyperframes turns HTML into video.
  <break time="350ms"/>
  <emphasis level="moderate">Same input, identical output.</emphasis>
</speak>
```

### Step 4 — Place the produced `.wav` into the HyperFrames project

```html
<audio
  id="narration"
  data-start="0"
  data-duration="auto"
  data-track-index="2"
  src="narration.wav"
  data-volume="1"
></audio>
```

Use `data-duration="auto"` to let HyperFrames read the actual file duration via `ffprobe` at parse time. If the composition needs the duration declared explicitly, run `ffprobe -i narration.wav -show_entries format=duration -v quiet -of csv="p=0"` and substitute the result.

### Step 5 — Transcribe for caption alignment

Because `generate_speech` does not return word-level timestamps, you MUST transcribe the generated audio using the OpenAI Whisper API to get precise timing for captions and beat alignment.

Write a short Python script to call the API:

```python
from openai import OpenAI
client = OpenAI()
with open("narration.wav", "rb") as audio_file:
    transcript = client.audio.transcriptions.create(
        model="whisper-1", file=audio_file,
        response_format="verbose_json",
        timestamp_granularities=["word"]
    )
```

Convert the returned `transcript.words` into the `[{ text, start, end }]` shape. **Save this array as `transcript.json` in the project root** so downstream steps (caption grouping per `references/hyperframes/references/captions.md`, beat timing in STORYBOARD.md) can use it.

## Music generation contract

### Step 1 — Decide music role and arc

Before calling `generate_music`, decide:

- **Role** — underscore (low, beneath narration), full-energy bed (drives the piece), or transitional sting (3–5 s).
- **Mood arc** — does the energy peak at the climax beat? Drop at the CTA? Hold flat across a tutorial? The `music-prompter` skill is authoritative on prompt construction.
- **Duration** — exactly the composition duration plus 0.5 s tail; `generate_music` does not auto-trim.

### Step 2 — Call `generate_music` with full prompt

Use the 9-dimension prompt structure from the music-prompter skill. Save the output as `music.wav` (or `underscore.mp3` for the website-to-hyperframes step-6 convention).

### Step 3 — Mix the music track into the project

```html
<audio
  id="bg-music"
  data-start="0"
  data-duration="auto"
  data-track-index="3"
  src="music.wav"
  data-volume="0.5"
></audio>
```

`data-volume` defaults: **0.10–0.20** under narration, **0.50–0.70** when music is the lead, **1.0** for transitional stings.

### Step 4 — Audio-reactive driving

If a beat needs to react to the music (per `references/hyperframes/references/audio-reactive.md`), pre-extract the audio data with the upstream script `references/gsap/scripts/extract-audio-data.py` and feed the array into the GSAP timeline as documented there. The script reads any local `.wav` / `.mp3` and is unaffected by Manus / upstream routing.

## Image generation contract

### When to generate

Generate raster assets with `generate_image` whenever the composition needs:

- A product/marketing image that the user did not upload;
- An icon, logo lockup, or symbolic illustration not available in `assets/`;
- A background plate (gradient art, photographic texture, abstract pattern) that motion will operate on;
- Variations of an existing image (use `generate_image_variation`).

### When **not** to generate

Do not generate images when the composition only needs:

- Pure CSS / GSAP shapes — let the framework draw them.
- Iconography that already lives in the registry (`hyperframes add <name>`).
- Charts / graphs — generate them as live SVG / Canvas inside the composition; do not produce them as a flat PNG.

### Aspect ratio and resolution

Match the composition's output frame: **16:9 (1920×1080)** or **9:16 (1080×1920)** by default. Square (1080×1080) only when the storyboard explicitly calls for an Instagram feed asset. Never request 1:1 or other ratios for use as full-frame backgrounds.

### Place the produced PNG into the project

```html
<img
  id="hero"
  class="clip"
  data-start="0"
  data-duration="5"
  data-track-index="1"
  src="hero.png"
/>
```

Per `references/hyperframes/references/motion-principles.md`, every image must have motion treatment (perspective tilt, slow zoom, device frame, floating UI, scroll reveal). Never embed a raw flat image.

## External-audio transcription contract

When the user supplies external audio that did not come from `generate_speech`, use the same OpenAI Whisper API approach as Step 5 above to get word-level timestamps.

Convert the returned `transcript.words` into the `[{ text, start, end }]` shape, save it as `transcript.json`, and pass to caption grouping per `references/hyperframes/references/captions.md`. Do not run `npx hyperframes transcribe` or whisper.cpp.
